Caracteristiques techniques Oguiz se doit de révolutionner aussi bien la vie des professionnels du loisir que de leur consommateurs
pour se faire Oguiz devra disposer d'une architecture particuliere :

serveurs :

serveur asterisk (telephonie)

serveur de données

API :
Gocardless (paiement mobile)

google speech2text (analyse de la voix)

smsmode (envoi de sms)

de plus afin de facilité la vie de nos chère développeurs une architecture de test/production a été mis en place:

storybook (test des composant)
git (code versionning)
trello
confluence
phpunit (test de code)
git webhook (CI/CD)
mise à jour et maintenance afin d'assurer le bon fonctionnement de l'application sans interruption des Outils de CI/CD ont ete mis en place dont:
-un service de versionning de code afin que chaque ajout/modification de code soit historisé et reversible (les merge request seront accepté par Mr Machroub),
a chaque merge request acceptée, les changements seront non seulement uploader sur le git lais aussi sur un serveur de test.
-des outils de monitoring  ont aussi ete mis en place (sentry,outil de ticketing from scratch...) afin de verifier le bon fonctionnement du code a tout moment
